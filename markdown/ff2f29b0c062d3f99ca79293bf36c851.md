# Agent based modelling
Last Updated : 23 Jul, 2025
Agent based modelling is a class of computational models that simulate the behaviour of autonomous agents such as people, animals, vehicles or organizations, or even cells. Each agent operates according to a set of rules and interacts with other agents and their environment and then the model then tracks how these local interactions give rise to global patterns.
![agents_⟨44⟩]Agent based modelling
## Agents
  * In agent based modelling (ABM) agents are autonomous, decision making entities that simulate the behavior of individuals components within a system. 
  * Each agent operates according to a set of rules that govern its behavior, decision making processes and interactions with other agents and the environment. 
  * Agents are typically heterogeneous as they can differ in attributes such as age, preferences, goals, location or memory. 
  * Agents can be reactive responding to change in their environment or proactive acting with intention or purpose based on internal states or goals. Some models include learning agents which adapt their behavior over time using mechanisms like reinforcement learning⟨45⟩, genetic algorithms or evolutionary strategies. 


## Components of Agent based models
  1. ****Agents**** : Independent entities with their own attributes and behavioral rules.
  2. ****Environment**** : The space where agents exist and interact.
  3. ****Rules of Interaction**** : Guidelines that define how agents interact with each other and their surroundings.
  4. ****Emergence**** : The system level patterns or outcomes that arise from simple interactions among agents.


## Importance of Agent based models
  1. ****Captures Complexity**** : Agent based modeling is a technique used for representing complex systems where things don’t follow a straight path. In such systems individual agents interact in ways that create nonlinear effects, feedback loops, and adaptive behavior.
  2. ****Bottom up approach**** : Instead of assuming overall system behavior from the top ABM works from the bottom up. It starts by modelling the individual behaviors of agents. As these agents interact, the overall system behavior emerges naturally.
  3. ****Flexibility**** : ABM is highly flexible as it can be used to model both spatial systems and non spatial systems and it supports simulations over both discrete time steps and continuous time which makes it suitable for a wide range of fields and problems.
  4. ****Scenario Testing**** : ABM allows you to run 'what if' scenarios which helps researchers, businesses, and policymakers test decisions in a virtual environment before applying them in the real world which saves time, money and reduce risk.


## Challenges in ABM
  1. ****Data requirements**** : one of the biggest challenges in creating an effective ABM is the need for detailed and accurate data about how individual agents behave as it works best when it can simulate how people, machines or organizations act in real life. 
  2. ****Computational intensity**** : agent based models with thousands or millions of agents can be computationally demanding as every agent acts individually and interacts with others the system must keep track of a lot of variables at once which requires powerful hardware and can take a long time to run simulations.
  3. ****Design sensitivity**** : agent based models can be very sensitive to small changes in design and the model needs to be tested and interpreted carefully. Otherwise small modelling errors could lead to wrong conclusions.
